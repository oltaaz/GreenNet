This is the top-level submission guide for the repository.

2. Read [docs/final_submission_overview.md](/Users/enionismaili/Desktop/GreenNet/docs/final_submission_overview.md).
3. Open [artifacts/final_pipeline/official_acceptance_v1/README.md](/Users/enionismaili/Desktop/GreenNet/artifacts/final_pipeline/official_acceptance_v1/README.md).
4. If you want the final report directly, open [artifacts/final_pipeline/official_acceptance_v1/summary/final_evaluation/final_evaluation_report.md](/Users/enionismaili/Desktop/GreenNet/artifacts/final_pipeline/official_acceptance_v1/summary/final_evaluation/final_evaluation_report.md).

Canonical reviewer-facing evidence:

- benchmark definition: `configs/acceptance_matrices/official_acceptance_v1.json`
- canonical final bundle: `artifacts/final_pipeline/official_acceptance_v1/`
- reviewer entry file: `artifacts/final_pipeline/official_acceptance_v1/report/reviewer_start_here.md`
- structured store: `artifacts/db/greennet.sqlite3`
- public UI: `frontend/greennet-ui/`

Historical or internal paths that are intentionally not the primary submission path:

- `_exports/` contains archival export snapshots
- `results/` contains operational and historical run folders
- `tmp/official_acceptance_backup/` is a preserved backup area
- `experiments/official_matrix_v1` through `experiments/official_matrix_v6` are preserved historical experiment bundles
- `artifacts/final_pipeline/latest/` is an archival pinned historical bundle, not the canonical final evidence path
- `final_audit/` contains internal audit and repair evidence, not the thesis-facing benchmark bundle itself

If a path is not listed above as canonical reviewer-facing evidence, treat it as supporting, historical, or internal material unless a document explicitly says otherwise.
